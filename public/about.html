About

This is the about page of the Phone Book Application.

Functionality

This is a simple, yet fully functional application that allows insertion and deletion of phone book records to and from the database. Another functionality offered is filtering of database records agains the last name field. However, I was free to enrich the filter functionality in such a way that it allows filtering against more than one last name, and also filtering against last names that begin or end with a specified substring. For example if we specify a filter like "Pa...", app will return all the records where last name begins with "Pa" (like "Pantic", "Paunovic".... Similarly, if we specify a filter like "...ic", app will return all records where last name ends with "ic" (like "Smiljanic", "Pantic"...). If we want to specify more than one last name criteria, we must delimit them with semicolon character(";"). for example: "Djurkov;Pa...;...ic" will get all records where last name is "Djurkov" or last name begins with "Pa" or last name ends with "ic".

Insertion of records into the database is done by submitting a manually populates form. Form validates entries on the client side and allows submissio only if all the fields are populated with valid data. Upon insertion of record, phone book is rerendered with added information about the inserted record.

Deletion of records from the database is implemented so that it can delete multiple entries at once. Each listed record in the phone book has a checkbox that can be checked. After interacting with "Dele selected" submit button, all selected records will be deleted from the database and the phone book is rerendered with added list of deleted records.

Database consists of two tables: "persons" and "phone_numbers". It was designed like so in order to eliminate redundancy of first_name and last_name entries, because one person can have many phone numbers (person -> phone_number = one-to-many relationship). Both insertions of records to and deletions from the database are performed on both tables and are doing checks in order to avoid corruption of tables consistency. For example, if a user is inserting a phone number for a person that is already present in the database, only phone_number will be inserted into "phone_numbers" table. Otherwise, new person will be added to the "persons" table, and phone number will be added to the "phone_numbers" table.
